# Performance Baselines

This file is the single source-of-truth for workspace performance defaults.

## Policy
- Keep heavyweight environments/artifacts outside workspace root when feasible.
- Do not duplicate exclusion/policy tables in other docs; reference this file.
- Never delete old environments before replacement is verified.

## Diagnosis First
1. Run `code --status` when VS Code is available.
2. Confirm whether watcher/indexing load is the bottleneck.

## Workspace Excludes Baseline

### File watcher excludes
| Pattern | Reason | Stack |
|---|---|---|
| `**/.git/**` | Avoid repository object churn | all |
| `**/.venv/**` | Python environment heavy file count | python |
| `**/__pycache__/**` | Generated bytecode | python |
| `**/.pytest_cache/**` | Test cache | python |
| `**/.mypy_cache/**` | Type-check cache | python |
| `**/build/**` | Build outputs | python/tooling |
| `**/dist/**` | Distribution artifacts | python/tooling |
| `**/*_run/**` | Generated run artifacts | app-specific |

### Search excludes
| Pattern | Reason |
|---|---|
| `**/.venv/**` | Reduce noisy search results and scan time |
| `**/dist/**` | Generated packaging output |
| `**/build/**` | Generated build output |
| `**/*_run/**` | Runtime artifacts |

## OS-Conditional Guidance
- Windows: consider Defender exclusions for verified high-churn generated directories.
- macOS/Linux: use platform indexing exclusions for generated directories.

## Stack-Conditional Notes
- Python repos: include `.venv`, `__pycache__`, `.pytest_cache`, `.mypy_cache`.
- Add Node/Flutter exclusions only if those stacks are in build/test path.
